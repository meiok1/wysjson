# wysJSON

`wysJSON` 当前是一个单文件 JSON 表格编辑器原型，主文件是 `index.html`。

项目目标不是先做完整工程化，而是先把下面这件事做扎实：

- 用接近 Excel 的交互方式编辑 JSON
- 支持 object / array / primitive 混合嵌套
- 尽量通过 UI 完成节点增删改和类型切换
- 为未来封装成 VS Code Webview 保留基础

## 当前入口

- 主页面：`index.html`
- 详细项目记忆体：`wysLog/README.md`

如果你是新开会话的大模型，建议按这个顺序理解项目：

1. 先读 `wysLog/README.md`
2. 再读 `index.html`
3. 改动前先复述当前规则，再动手

## 当前真实状态

这是一个纯前端原型，不是完整 VS Code 扩展。

当前没有构建步骤，也没有 npm 依赖。运行方式很简单：

- 直接在浏览器打开 `index.html`
- 或用静态服务器打开 `index.html`

注意：

- 不要用 `node --check index.html` 判断页面脚本是否正常
- 真正有效的验证方式是浏览器交互验证或 Playwright 验证

## 当前已确认的重要规则

- 只有纯 object 数组才走列式表格
- 混合数组必须逐行递归显示
- cell 右键菜单改的是当前值节点类型，不是整个父表
- corner 菜单是当前表/选区级别的节点操作
- `单值` 的规则是递归取第一个元素或第一个字段的值

## 当前最重要的文件

- `index.html`
  当前全部功能都在这里，包含 HTML / CSS / JS / 状态 / 渲染 / 菜单逻辑。

- `wysLog/README.md`
  这是详细版项目记忆体，记录当前架构、规则、验证样例、踩坑点、协作方式。
