# wysJSON VS Code Extension - 测试与部署计划

### 核心功能模块
- ✅ **项目结构** - TypeScript + Babel 设置完整

- ✅ **Table UI** - `webview.js` + `webview.css`
  - Excel 风格的表格编辑
  - 嵌套数据展开/收起

## 🧪 测试步骤（本地集成测试）

### 1. 启动 Extension Development Host
```bash
# 在项目根目录
npm run compile    # 确保编译成功
# 然后按 F5 启动 Extension Development Host
```

### 2. 测试简单 JSON 编辑
- 在 DEV Host 中打开 `jsonDemo/testData.js`
- 选择 `simpleData` 对象定义（包括 `const simpleData = { ... }` 的整个语句）
- 右键选择 "wysJSON: 可视化编辑所选 JS 数据"
- ✓ Webview 应显示表格，包含 name, age, active, email 字段
- ✓ 编辑某个值（如改 age 为 31）
- ✓ 点击"💾 保存"
- ✓ 验证原文件中 age 已更新为 31

### 3. 测试嵌套数据编辑
- 选择 `nestedData` 对象
- 右键打开 wysJSON
- ✓ 应显示 users（数组）和 meta（对象）两个主要字段
- 点击 users 行展开数组
- ✓ 应显示 3 条记录，每条有 id, name, email
- 编辑第二条记录的 name（Bob → Bobby）
- 保存并验证

### 4. 测试 codeText 节点处理
- 选择 `complexData` 对象
- 右键打开 wysJSON
- ✓ 应显示各个字段：
  - name: 正常文本
  - created: `codeText` - 显示 "new Date(...)"
  - handler: `codeText` - 显示 "function(item) { ... }"
  - items: 数组，可编辑
  - undefined_field: `codeText` - 显示 "undefined"
- 编辑 items 中的数字值（如 100 → 150）
- 保存并验证 handler 函数保持原样

### 5. 测试数组编辑
- 选择 `simpleArray`
- 右键打开 wysJSON
- ✓ 应显示 5 个值，带行号
- 编辑某个值（apple → apricot）
- 保存并验证

### 6. 容错测试
- **多个 literal**：选择包含 2 个对象定义的文本，应提示错误
- **非 literal**：选择函数调用 `console.log(...)` 应无法打开
- **混合类型**：选择只包含函数定义的行应提示无法识别

## 🐛 已知问题与限制

### 当前版本限制
1. **只读代码节点** - 包含函数/Symbol/undefined 的节点显示为代码文本，不能在 UI 中编辑代码内容
   - 解决方案：在 webview 中添加代码编辑器（下个版本）
2. **单级选择** - 仅提取选区内唯一的 object/array literal
   - 不支持嵌套选择部分结构
3. **无重做/撤销** - Webview 中的编辑撤销不同步到扩展
   - 解决方案：实现完整的历史堆栈（下个版本）

### 边界情况
- 包含注释的对象定义 - 注释会被解析器移除
- 使用 Map/Set/WeakMap 的对象 - 显示为 codeText（无法序列化）
- 包含 getter/setter 的对象 - 显示为对象方法

## 📦 部署与发布

### 本地打包
```bash
npm install -g vsce  # 一次性安装打包工具
vsce package         # 生成 .vsix 文件
```

### 测试部署
```bash
# 在 VS Code 中本地安装扩展
code --install-extension wysjson-0.1.0.vsix
```

### 发布到 VS Code Marketplace
```bash
vsce publish       # 需要 publisher token
```
